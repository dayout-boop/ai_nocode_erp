## 1. 처음 세 가지 질문에 대한 최종 답

### Q1. 마누스 API 연결을 ERP 내에서 하면 다른 서버로 이전해도 문제없나?

**이제 "예"라고 답할 수 있습니다.** 다만 핵심은 두 가지를 모두 해결했다는 점입니다.

- **(키)** 마누스/OpenRouter/Anthropic/S3/검색 키를 ERP 설정 화면에서 등록·암호화 저장 → 어느 서버든 DB만 따라가면 키가 유지됩니다.
- **(호출 경로)** 키만으로는 부족했던 부분 — 마누스 서비스(`forge.manus.im` 등)에 직접 호출하던 기능들에 **대체 경로(폴백)**를 넣었습니다. 이제 마누스가 없어도 OpenRouter·S3·Slack·Serper 등으로 자동 전환됩니다.

즉 **"키 + 호출 경로" 양쪽을 모두 자립화**했기 때문에, 다른 서버로 옮겨도 ERP 설정만 맞추면 동작합니다.

### Q2. invokeLLM / manusPipe / manusSync 변경 시 두골프마스터·매니저·골프톡에 문제가 생기나?

**생기지 않습니다.** 코드 전수조사로 교차검증했습니다.

- 채팅 3종(두골프마스터·매니저·골프톡)의 **대화 본체는 이미 OpenRouter 직결**이라 `invokeLLM`/`manusPipe`/`manusSync`를 경유하지 않습니다.
- 유일한 예외였던 **두골프마스터의 웹검색 도구**도 이번에 Serper 폴백을 넣어, 마누스 없이도 검색 키만 등록하면 동작하고, 없어도 빈 결과로 graceful 처리되어 대화는 끊기지 않습니다.
- **응답 지연 우려**: 폴백은 1순위 실패 시에만 동작하므로 정상 상황에서 추가 지연이 없습니다.

### Q3. 기존 방식과 어느 서버로 옮겨가도 둘 다 적용 가능하게 — 추천 방향은?

**채택한 방향: 듀얼 폴백(이중화) + ERP 제공자 선택.**

- 기본값(`auto`)은 기존과 100% 동일하게 마누스 forge 우선 동작
- ERP 설정에서 `OpenRouter 우선` 또는 `forge 전용`으로 즉시 전환 가능
- 1순위 장애 시 자동으로 2순위로 폴백 → 서비스 무중단

## 4. 남은 권장 사항 (선택)

1. **공개 홈페이지 일반회원 로그인**은 여전히 마누스 OAuth(`api.manus.im`)에 의존합니다. ERP 마스터/직원 로그인은 자체 인증이라 무관하지만, 일반 회원 로그인까지 자립이 필요하면 자체/구글 OAuth 도입을 별도 검토할 수 있습니다.
2. 다른 서버로 실제 이전 시 ERP 설정에서 **OpenRouter 키 + S3 설정 + (선택)Serper 키**를 등록하고, 기본 LLM 제공자를 `OpenRouter 우선`으로 전환하면 됩니다.

## 5. 결론

마스터님의 우려대로 처음 분석(LLM 3개)은 부분적이었고, 전수조사 결과 **스토리지가 LLM보다 더 큰 숨은 종속**이었습니다. 이번 작업으로 LLM·스토리지·개발파이프·웹검색·알림·관리자세션까지 **모두 듀얼 폴백 또는 자립 구조로 전환**했으며, 기존 방식(마누스 환경)은 그대로 보존됩니다. 어느 서버로 옮겨가도 ERP 설정만으로 양쪽 다 동작합니다.
