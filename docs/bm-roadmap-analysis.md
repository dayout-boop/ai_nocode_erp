# 두골프 AI 플랫폼 BM 전략 분석 (2026-04-30 문서 기반)

## 핵심 4개 엔진
- AI 개발 엔진: 코드 자동 생성, 기능 추가, 버그 수정 (Manus API 연동 완료)
- AI 마스터: 골프 상품 기획, 고객 상담, 콘텐츠 생성 (Gemini + OpenRouter 연동 완료)
- AI 에이전트: 개별 업무 자율 수행 예약/결제/알림 (n8n 자동화 연동 완료)
- AI 오케스트라: 다중 에이전트 협업 조율, 비용 최적화 라우팅 (설계 단계)

## 수익 모델
- 레이어1: 월 고정 서비스 이용료 (₩99,000~₩499,000/월, Basic/Pro/Enterprise)
- 레이어2: 사용량 기반 과금 (LLM 토큰, 이미지/영상 생성, 알림톡, 결제 중개)
- 레이어3: 연동 서비스 중개 수수료 (Stripe, 포트원, 카카오 알림톡)

## 비용 최적화 라우팅
- 텍스트 요약/분류: Gemini 2.0 Flash ($0.10/1M tokens)
- 고객 상담/콘텐츠 생성: Claude 3.5 Haiku ($0.80/1M tokens)
- 복잡한 분석/전략 수립: Gemini 2.5 Pro ($1.25/1M tokens)
- 코드 생성/디버깅: Manus API (크레딧 기반)
- Prompt Caching 적용 시 반복 시스템 프롬프트 비용 최대 90% 절감

## 단기 개발 우선순위 (다음 4주)
1. AI 오케스트라 라우터 구현 — OpenRouter 기반 작업 복잡도별 모델 자동 선택
2. 사용자 온보딩 자동화 — 신규 가입 시 n8n 워크플로우 자동 실행
3. 결제 일괄 조회 연동 — 토스 오픈뱅킹 또는 Plaid로 실제 구독 비용 자동 파악
4. API 키 암호화 저장 — 파트너별 서비스 API 키를 AES-256으로 암호화하여 DB 저장
5. 자기학습 피드백 루프 — AI 호출 결과 자동 평가 및 모델 선택 최적화

## 신규 파트너 온보딩 프로세스 (사용자 요청 기반 추가 분석)
- 두골프 매니져: 신규 사업자 간편가입 (구글/네이버 로그인) + 사업자등록증 OCR + 통장사본 확인 + 무료/유료 플랜 선택 + 가입심사
- 가입심사 후: 카테고리별 샘플 선택 (여행사/골프장/쇼핑몰/실내연습장/숙소)
- 샘플 선택 후: 현재 두골프 ERP/홈페이지를 복제하여 개인정보 제거 후 초기 세팅
- {사업체 브랜드명} 매니져 생성: 해당 파트너 전용 챗봇 (다른 프로젝트 DB 접근 불가)
- Manus 신규 프로젝트 자동 생성 + 두골프 마스터가 하위 프로젝트로 자동 관리

## 아키텍처 계층 구조
```
두골프 마스터 (전체 관리)
├── 두골프 ERP/홈페이지 (현재 프로젝트)
│   └── 골프톡 (고객 챗봇)
├── {파트너A 브랜드} ERP/홈페이지
│   ├── {파트너A} 매니져 (파트너 전용 챗봇)
│   └── {파트너A}톡 (고객 챗봇)
└── {파트너B 브랜드} ERP/홈페이지
    ├── {파트너B} 매니져
    └── {파트너B}톡
```
